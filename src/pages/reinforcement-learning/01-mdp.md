# 強化学習とは — 問題設定とマルコフ決定過程 (MDP)

:::abstract[学習目標]
この章を読み終えると、次のことができるようになります。

- 強化学習の問題を **エージェント-環境ループ**（状態 $S$・行動 $A$・報酬 $R$）として説明できる
- **方策 $\pi$**・**収益 $G_t$**・**割引率 $\gamma$**・**価値関数 $V^\pi$ / $Q^\pi$** をすべて定義し、互いの関係を式で書ける
- 問題設定を **マルコフ決定過程 (MDP)** の5要素 $(\mathcal{S}, \mathcal{A}, P, R, \gamma)$ として定式化し、**マルコフ性**が何を意味するかを述べられる
- **ベルマン期待方程式**を導出し、価値関数が満たす再帰的整合条件として説明できる
- numpy で小さな gridworld MDP を作り、ランダム方策の収益をシミュレートして $V^\pi$ を推定し、**厳密解と一致する**ことを確かめられる
- 報酬 $R$ と価値 $V$ の違い、RL が他モダリティ（RLHF・身体制御）から参照される理由を説明できる
:::

## 前提知識

- 確率・期待値の基礎：条件付き確率 $P(X\mid Y)$、期待値 $\mathbb{E}[\cdot]$、無限等比級数の和
- 線形代数の基礎：行列とベクトルの積、連立一次方程式 $A\mathbf{x}=\mathbf{b}$ を解くこと
- Python と numpy の基礎（実装節）
- 分野の俯瞰：[強化学習ロードマップ](/reinforcement-learning/)（この軸が各モダリティから参照される理由）

この章は強化学習シリーズの最初の1本です。前提として RL の知識は要りません。一から積み上げます。LLM 出身の読者には、随所で **自己回帰生成とのアナロジー**（次トークン予測 ↔ 次行動選択）を橋渡しに使います。

## 直感

教師あり学習は「正解ラベル付きの例」から学びます。画像にネコと書いてあれば、それが正解です。ところが、世の中には **「正解が一手ずつは与えられない」問題** があります。

- 囲碁で「いま打つべき一手」の正解は誰も教えてくれません。分かるのは **対局が終わったときの勝敗** だけです。
- ロボットアームに「いまモーターを何度回せ」の正解はありません。分かるのは **最終的に物をつかめたか** だけです。
- LLM に「この単語の次に出すべき最良の単語」を人手で全部ラベル付けはできません。分かるのは **出力全体が好まれたか** だけです。

これらに共通するのは、**遅れて・まばらにしか来ない手がかり（報酬）を頼りに、一連の行動を最適化する** という構造です。これを **逐次決定問題 (sequential decision making)** と呼びます。強化学習 (Reinforcement Learning, RL) は、この問題を解く枠組みです。

:::tip[RL は「モダリティ」ではなく「学び方」]
画像・音声・テキストは入出力の **様式（モダリティ）** です。一方 RL は、どんな様式の問題であっても使える **学習パラダイム** です。だから RL は他のモダリティから繰り返し参照される **横断軸** になります。言語では RLHF、身体性では学習ベース制御 —— 中身はどれも「報酬で逐次決定を最適化する」同じ言語で書けます。この章は、その共通言語の **文法（MDP）** を定義する回です。
:::

ゴールはシンプルです。**報酬の累積（収益）を最大にするような行動の選び方（方策）を、環境との試行錯誤から学ぶ。** この一文を、誤解なく・式で書けるところまで分解していきます。

## 全体像

強化学習のすべては、次の **エージェント-環境ループ** に収まります。順方向（行動を出す）と逆方向（結果が返る）を1枚で押さえます。

```mermaid
flowchart LR
  AG["エージェント<br/>方策 π(a|s)"] -->|"行動 A_t"| ENV["環境<br/>遷移 P・報酬 R"]
  ENV -->|"次状態 S_{t+1}"| AG
  ENV -->|"報酬 R_{t+1}"| AG
```

時刻 $t$ ごとに、次の3つが順に起きます。

1. エージェントは現在の **状態 (state) $S_t$** を観測する。
2. 方策 $\pi$ に従って **行動 (action) $A_t$** を選び、環境に渡す。
3. 環境は **次状態 $S_{t+1}$** と **報酬 (reward) $R_{t+1}$** を返す。

これを $t=0,1,2,\dots$ と繰り返して生まれる列

$$
S_0,\, A_0,\, R_1,\, S_1,\, A_1,\, R_2,\, S_2,\, \dots
$$

を **軌跡 (trajectory)** と呼びます。学習とは、この軌跡から得る報酬の累積を大きくするように、方策 $\pi$ を調整することです。

:::note[LLM ↔ RL のアナロジー]
LLM の自己回帰生成と構造が同じです。**状態 $S_t$ = これまでの文脈**、**行動 $A_t$ = 次に出すトークン**、**方策 $\pi(a\mid s)$ = 次トークン分布**。違いは1点 —— LLM の事前学習は「次トークンの正解（教師）」で学びますが、RL は **正解の代わりに報酬** で学びます。RLHF（章 [LLM 適応と RLHF](/llm/05-adaptation-rlhf/)）は、まさにこの LLM をエージェントとみなして報酬で方策を最適化する応用です。
:::

この章で辿る構造は次の通りです。**ループ → 5つの量を定義（$S, A, R, \pi, G$）→ それらを束ねる MDP → 価値関数 $V/Q$ → 価値が満たすベルマン方程式 → numpy で確かめる。** 用語の地図を先に置きます。

| 記号 | 名前 | 一言でいうと |
| --- | --- | --- |
| $S_t$ | 状態 | いま環境がどうなっているか |
| $A_t$ | 行動 | エージェントが打つ手 |
| $R_{t+1}$ | 報酬 | その手の直後にもらう即時の点数 |
| $\pi(a\mid s)$ | 方策 | 状態を見て行動を選ぶ規則（確率分布） |
| $G_t$ | 収益（return） | 時刻 $t$ 以降の **割引報酬の総和** |
| $\gamma$ | 割引率 | 将来の報酬をどれだけ重く見るか（$0\le\gamma\le1$） |
| $V^\pi(s)$ | 状態価値 | 状態 $s$ から方策 $\pi$ で得られる収益の期待値 |
| $Q^\pi(s,a)$ | 行動価値 | $s$ で行動 $a$ を取り以後 $\pi$ に従う収益の期待値 |

## 理論

ここから、上の各量を §2 の深さ基準に従って正確に定義します。記号は **何から作るか・何を見て何を見ないか・どの軸でインデックスされるか・役割** まで書きます。

### 状態・行動・報酬

**状態 $S_t \in \mathcal{S}$。** 時刻 $t$ における環境の記述です。$\mathcal{S}$ は取りうる状態の集合（状態空間）。gridworld なら「いまどのマスにいるか」、囲碁なら「盤面」、LLM なら「これまでのトークン列」。エージェントが意思決定に使える情報の全体です。**何を見るか** が状態の設計そのもので、ここに「未来を決めるのに必要な情報」が入っているかどうかが後述のマルコフ性を左右します。

**行動 $A_t \in \mathcal{A}$。** エージェントが時刻 $t$ で選ぶ手です。$\mathcal{A}$ は行動空間。**離散**（gridworld の上下左右、LLM の語彙）と **連続**（ロボットの関節トルク）があり、この区別が後の章でアルゴリズムを分けます（離散は価値ベース Q 学習が、連続は方策ベースが向く）。

**報酬 $R_{t+1} \in \mathbb{R}$。** 行動 $A_t$ を取った **直後** に環境が返す **スカラーの即時評価** です。添字に注意：$A_t$ への報酬は $R_{t+1}$ と書く慣習です（行動を取ってから1ステップ後に観測されるため）。報酬は **設計者が与える**もので、何を報酬にするかが「何を学ぶか」を決めます。

:::warning[添字のズレ：$R_t$ か $R_{t+1}$ か]
本章では Sutton & Barto の標準に従い、状態 $S_t$ で行動 $A_t$ を取った結果を $R_{t+1}, S_{t+1}$ と書きます。つまり **報酬の添字は「行動した時刻 + 1」**。文献によっては同じものを $R_t$ と書くものもあります。式を読むときは「報酬が行動の前か後か」を必ず確認してください。一手ズレると収益の定義式が変わって見えます。
:::

### 方策 $\pi$：状態から行動への規則

**方策 (policy) $\pi$** は、状態を見て行動を選ぶ規則です。**確率的方策**は条件付き分布

$$
\pi(a\mid s) = P(A_t = a \mid S_t = s), \qquad \sum_{a}\pi(a\mid s)=1
$$

として書きます。**何から作るか**：現在の状態 $s$ だけ（過去の全履歴ではなく）。**役割**：エージェントの「脳」そのもの。学習とは、この $\pi$ を良くすることです。決定論的方策 $a=\mu(s)$ は、ある行動に確率 1 を割り当てる特別な場合です。

:::note[LLM ↔ RL のアナロジー]
$\pi(a\mid s)$ は LLM の **次トークン分布 $p(\text{token}\mid \text{context})$** と完全に同じ型です。状態が文脈、行動がトークン、方策が出力分布。RL で「方策を最適化する」とは、LLM で「次トークン分布を望ましい方向に動かす」のと同じ操作です。
:::

### 収益 $G_t$：将来の割引報酬の総和

エージェントが本当に最大化したいのは、**1回の報酬ではなく、これから先の報酬の累積** です。時刻 $t$ から見た **収益 (return) $G_t$** を、割引率 $\gamma$ を使って次で定義します。

$$
G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^{k} R_{t+k+1}
$$

- **$\gamma \in [0,1]$：割引率 (discount factor)。** 将来の報酬をどれだけ重く見るか。$\gamma=0$ なら「目先の $R_{t+1}$ だけ見る近視眼」、$\gamma \to 1$ なら「遠い将来まで等しく見る遠視眼」。$k$ ステップ先の報酬は $\gamma^k$ 倍に縮みます。
- **なぜ割り引くのか（why）。** 3つ理由があります。(1) 数学的に、報酬が有界なら $\gamma<1$ で無限和が収束する（発散しない）。(2) 不確実な遠い将来より、確実な目先を重く見るのは自然。(3) 「いま貰える1点は将来の1点より価値がある」という時間選好を表せる。
- **$G_t$ は確率変数です。** 同じ状態から始めても、方策の確率性や環境の確率性で軌跡が変わり、$G_t$ も毎回変わります。だから次節で **期待値** を取ります。

:::warning[報酬 $R$ と価値 $V$ は別物 —— 最重要の誤解]
ここが最初の関門です。混同しないでください。

| | 報酬 $R_{t+1}$ | 価値 $V^\pi(s)$ |
| --- | --- | --- |
| いつの量か | **即時**（その一手の直後にもらう1スカラー） | **将来全体**（以後の収益の期待） |
| 誰が決めるか | 環境（設計者が報酬関数を与える） | 方策と環境から **計算/推定される** 量 |
| 一言で | 「いまの点数」 | 「ここから先、トータルでどれだけ稼げるか」 |

gridworld では毎歩 $-1$ の報酬でも、ゴール近くの状態は価値が高く（あと少しで終わる＝損失が小さい）、遠い状態は価値が低い（まだ多くの $-1$ を払う）。**報酬は一定でも価値は場所で違う。** 「報酬の高い行動」を貪欲に選ぶのではなく「**価値**の高い状態へ向かう行動」を選ぶ —— ここが RL の本質です。実装節で、この価値の地形を数値で見ます。
:::

### マルコフ決定過程 (MDP)：問題設定の器

ここまでの量（$S, A, R, \pi, G$）を1つの数学的な器に収めたものが **マルコフ決定過程 (Markov Decision Process, MDP)** です。MDP は **5つ組** $(\mathcal{S}, \mathcal{A}, P, R, \gamma)$ で定義されます。

- $\mathcal{S}$：状態空間。
- $\mathcal{A}$：行動空間。
- $P(s'\mid s,a) = P(S_{t+1}=s' \mid S_t=s, A_t=a)$：**遷移確率**。状態 $s$ で行動 $a$ を取ったとき次状態 $s'$ になる確率。環境のダイナミクスそのもの。
- $R(s,a)$ または $R(s,a,s')$：**報酬関数**。即時報酬の（期待）値。
- $\gamma$：割引率。

**MDP は「問題」を表し、方策 $\pi$ は「解（の候補）」を表します。** MDP は環境側が決め、$\pi$ はエージェント側が学ぶ。両者は別物です。

:::note[なぜ「決定 (Decision)」過程か]
行動の選択が入らない $(\mathcal{S}, P, R, \gamma)$ だけのものを **マルコフ連鎖／マルコフ報酬過程**と呼びます。そこにエージェントの **行動による選択（decision）** が加わると **マルコフ決定過程**になります。RL が扱うのはこの「行動で未来を変えられる」設定です。
:::

### マルコフ性：「現在の状態が未来に十分」

MDP の "M"、**マルコフ性 (Markov property)** は次の式が示す性質です。

$$
P(S_{t+1}\mid S_t, A_t, S_{t-1}, A_{t-1}, \dots, S_0, A_0) = P(S_{t+1}\mid S_t, A_t)
$$

**言葉でいうと**：次に何が起きるかは、**現在の状態 $S_t$ と行動 $A_t$ だけ** で決まり、それより前の履歴をいくら追加しても確率は変わらない。過去は現在の状態に **すべて畳み込まれている**、という主張です。

:::warning[マルコフ性は「未来が現在だけで決まる」ではなく「現在の状態が未来予測に十分」]
よくある誤解：「未来が完全に決まる（決定論）」という意味ではありません。マルコフ性は **確率分布** についての主張で、「次状態の **確率分布** は現在の状態と行動だけで決まる」と言っているだけです。次状態自体はランダムで構いません。

もう1つの誤解：「過去が無関係」ではありません。過去の情報が必要なら、**それを状態に含めればよい**のです。例：将来予測に速度が要るなら、状態を「位置」でなく「位置 + 速度」にすればマルコフ性が回復します。だから「マルコフ性が成り立つかどうか」は、本質的に **状態をどう設計したか** の問題です。「現在の状態が未来予測に十分な情報を持っている」—— これがマルコフ性の正しい読み方です。
:::

:::note[LLM ↔ RL のアナロジー]
LLM はまさにマルコフ的に設計されています。次トークン分布は **これまでの全トークン列（＝状態）** だけに依存し、それより前の「思考過程」のような隠れ履歴には依存しません。状態に文脈すべてを詰め込むことで、マルコフ性を満たしています。RL の「状態に必要な情報を全部入れる」発想と同じです。
:::

### 価値関数 $V^\pi$ と $Q^\pi$

収益 $G_t$ は確率変数でした。その **期待値** を取ると、方策の良さを評価できる決定論的な量になります。これが **価値関数 (value function)** です。

**状態価値関数 $V^\pi(s)$** は、状態 $s$ から **方策 $\pi$ に従い続けた** ときの収益の期待値です。

$$
V^\pi(s) = \mathbb{E}_\pi\!\left[G_t \mid S_t = s\right] = \mathbb{E}_\pi\!\left[\sum_{k=0}^{\infty}\gamma^k R_{t+k+1} \,\middle|\, S_t = s\right]
$$

- **何を見るか**：始点の状態 $s$ と、固定された方策 $\pi$。期待値 $\mathbb{E}_\pi$ は、$\pi$ が選ぶ行動と環境の遷移の両方のランダムさについて取ります。
- **役割**：「この状態は方策 $\pi$ のもとでどれだけ得か」を一数値で表す。状態に値札を貼る関数です。

**行動価値関数 $Q^\pi(s,a)$** は、状態 $s$ で **まず行動 $a$ を取り**、その後 $\pi$ に従ったときの収益の期待値です。

$$
Q^\pi(s,a) = \mathbb{E}_\pi\!\left[G_t \mid S_t = s,\, A_t = a\right]
$$

- $V^\pi$ との違いは **最初の一手** だけ。$V$ は「最初から $\pi$ に任せる」、$Q$ は「最初だけ $a$ を指定して、あとは $\pi$」。
- **役割**：「この状態でこの行動を取るのは得か」を測る。**行動の良し悪しを直接比較できる**ので、制御（より良い方策を作る）に直結します。次章以降の Q 学習はこの $Q$ を学びます。

両者は次の関係で結ばれます（方策で行動を平均すれば状態価値）。

$$
V^\pi(s) = \sum_{a}\pi(a\mid s)\,Q^\pi(s,a)
$$

実装節で、この等式が数値でぴったり成り立つことを確かめます。

## 数式の導出

ここでは **ベルマン期待方程式 (Bellman expectation equation)** を一から導きます。これは価値関数が満たす **再帰的な整合条件** で、次章以降のほぼすべてのアルゴリズム（動的計画法・TD 学習・Q 学習）の土台です。天下りせず、収益の定義から積み上げます。

**出発点：収益の再帰構造。** 収益の定義から、$G_t$ を「最初の報酬」と「残りの収益」に分けます。

$$
\begin{aligned}
G_t &= R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots \\
&= R_{t+1} + \gamma\,(R_{t+2} + \gamma R_{t+3} + \cdots) \\
&= R_{t+1} + \gamma\, G_{t+1}
\end{aligned}
$$

2行目では $\gamma$ で括り、3行目で括弧の中身が **$t+1$ から見た収益 $G_{t+1}$** そのものだと気づきます。これが鍵：**いまの収益 = 即時報酬 + 割り引いた次の収益。**

**状態価値に代入する。** $V^\pi(s) = \mathbb{E}_\pi[G_t \mid S_t=s]$ の $G_t$ を上の再帰で置き換えます。

$$
V^\pi(s) = \mathbb{E}_\pi\!\left[R_{t+1} + \gamma\,G_{t+1} \mid S_t = s\right]
$$

期待値の線形性で2項に分け、$\gamma$ を外に出します。

$$
V^\pi(s) = \underbrace{\mathbb{E}_\pi[R_{t+1}\mid S_t=s]}_{\text{即時報酬の期待}} + \gamma\,\underbrace{\mathbb{E}_\pi[G_{t+1}\mid S_t=s]}_{\text{次状態以降の期待収益}}
$$

**第2項を「次状態の価値」で書き直す。** $\mathbb{E}_\pi[G_{t+1}\mid S_t=s]$ を、起こりうる行動 $a$ と次状態 $s'$ で展開します。$s$ で $\pi$ に従い行動 $a$ を確率 $\pi(a\mid s)$ で選び、環境が確率 $P(s'\mid s,a)$ で $s'$ に遷移する。$s'$ に着いた後はマルコフ性により、それ以降の期待収益は $V^\pi(s')$ そのものです（$s'$ より前の履歴に依存しない）。よって

$$
V^\pi(s) = \sum_a \pi(a\mid s)\sum_{s'} P(s'\mid s,a)\Big[\,R(s,a,s') + \gamma\,V^\pi(s')\,\Big]
$$

これが **状態価値のベルマン期待方程式** です。読み方：**いまの状態の価値は、（取りうる行動と次状態すべてについて平均した）「即時報酬 + 割り引いた次状態の価値」に等しい。** 価値が自分自身を次状態経由で参照する **自己無撞着 (self-consistency) な等式** です。同様に行動価値については

$$
Q^\pi(s,a) = \sum_{s'} P(s'\mid s,a)\Big[\,R(s,a,s') + \gamma\sum_{a'}\pi(a'\mid s')\,Q^\pi(s',a')\,\Big]
$$

が成り立ちます。$\blacksquare$

:::tip[ベルマン方程式が線形方程式になる]
状態が有限個 $N$ なら、ベルマン期待方程式は **$N$ 本の連立一次方程式** です。ベクトル $\mathbf{V}\in\mathbb{R}^N$、方策のもとでの遷移行列 $\mathbf{P}_\pi$、期待即時報酬 $\mathbf{R}_\pi$ を使うと

$$
\mathbf{V} = \mathbf{R}_\pi + \gamma\,\mathbf{P}_\pi \mathbf{V}
\quad\Longrightarrow\quad
(\mathbf{I} - \gamma\,\mathbf{P}_\pi)\,\mathbf{V} = \mathbf{R}_\pi
$$

と行列形にまとまり、**$\mathbf{V} = (\mathbf{I} - \gamma\mathbf{P}_\pi)^{-1}\mathbf{R}_\pi$ と一発で解けます**（$\gamma<1$ なら逆行列は必ず存在）。実装節ではこれを「厳密解」として、モンテカルロ推定の答え合わせに使います。
:::

## 実装

小さな **4×4 gridworld の MDP** を numpy で定義し、(A) ランダム方策の収益をシミュレートして $V^\pi$ を **推定** し、(B) ベルマン期待方程式を線形に解いて **厳密値** を出し、両者が一致することを確かめます。学習というより「価値の定義が数値で何を意味するか」を体に入れる回です。

**設定**：4×4 の格子（状態 0〜15）。状態 15（右下）がゴール＝終端。行動は上下下右の4つ、壁にぶつかる行動はその場に留まる。報酬は **毎歩 $-1$**（最短でゴールを目指させる）、$\gamma=0.9$。方策は **各状態で4行動を等確率 $1/4$** のランダム方策。

```python title="gridworld_mdp.py"
import numpy as np

# ----- 4x4 gridworld の MDP 定義 -----
# 状態 s = 0..15（右下15 がゴール=終端状態）。行動 A = {0:上,1:下,2:左,3:右}。
# 壁にぶつかる行動はその場に留まる。報酬は毎歩 -1（最短ゴールを促す）。
GRID = 4
N_STATES = GRID * GRID
ACTIONS = [(-1, 0), (1, 0), (0, -1), (0, 1)]  # 上 下 左 右
GOAL = 15
GAMMA = 0.9  # 割引率 γ：将来をどれだけ重く見るか

def step(s, a):
    """状態 s で行動 a → (次状態 s', 即時報酬 r)。決定論的な環境ダイナミクス。"""
    if s == GOAL:
        return s, 0.0  # 終端：以後は留まり報酬 0
    r, c = divmod(s, GRID)
    dr, dc = ACTIONS[a]
    nr, nc = r + dr, c + dc
    if not (0 <= nr < GRID and 0 <= nc < GRID):
        nr, nc = r, c  # 壁：その場に留まる
    return nr * GRID + nc, -1.0  # 1 歩ごとに -1

rng = np.random.default_rng(0)
def sample_action(s):
    return rng.integers(0, 4)  # ランダム方策 π(a|s)=1/4

# ===== (A) モンテカルロで V^π(s) を推定 =====
# V^π(s)=E[G_t|S_t=s] の期待値を、s 始点の多数エピソードの収益 G の平均で近似する。
def run_episode(start, max_len=200):
    s, G, discount = start, 0.0, 1.0
    for _ in range(max_len):
        a = sample_action(s)
        s, r = step(s, a)
        G += discount * r       # 収益 G_t = Σ γ^k R_{t+k+1} を逐次累積
        discount *= GAMMA
        if s == GOAL:
            break
    return G

N_EPISODES = 20000
V_mc = np.zeros(N_STATES)
for s in range(N_STATES):
    if s == GOAL:
        continue  # 終端の価値は 0
    V_mc[s] = np.mean([run_episode(s) for _ in range(N_EPISODES)])

# ===== (B) ベルマン期待方程式を線形に解いて厳密な V^π を得る（答え合わせ）=====
# (I - γ P_π) V = R_π を numpy.linalg.solve で一発で解く。
P_pi = np.zeros((N_STATES, N_STATES))
R_pi = np.zeros(N_STATES)
for s in range(N_STATES):
    if s == GOAL:
        continue
    for a in range(4):
        s_next, r = step(s, a)
        P_pi[s, s_next] += 0.25   # π(a|s)=1/4 を遷移行列に積む
        R_pi[s] += 0.25 * r       # 期待即時報酬
V_exact = np.linalg.solve(np.eye(N_STATES) - GAMMA * P_pi, R_pi)

np.set_printoptions(precision=2, suppress=True)
print("V^pi (Monte Carlo 推定, 4x4):")
print(V_mc.reshape(GRID, GRID))
print("\nV^pi (Bellman を厳密に解いた値, 4x4):")
print(V_exact.reshape(GRID, GRID))
print(f"\nMC と厳密値の最大誤差: {np.max(np.abs(V_mc - V_exact)):.3f}")
```

実行結果（`uv run --with numpy python gridworld_mdp.py`）:

```text title="出力"
V^pi (Monte Carlo 推定, 4x4):
[[-9.37 -9.22 -8.98 -8.75]
 [-9.23 -8.96 -8.52 -7.99]
 [-8.99 -8.52 -7.41 -5.76]
 [-8.75 -7.94 -5.73  0.  ]]

V^pi (Bellman を厳密に解いた値, 4x4):
[[-9.36 -9.22 -8.97 -8.75]
 [-9.22 -8.97 -8.5  -7.97]
 [-8.97 -8.5  -7.42 -5.76]
 [-8.75 -7.97 -5.76  0.  ]]

MC と厳密値の最大誤差: 0.029
```

読み取り。**(1) 推定と厳密値が一致する**（最大誤差 0.029）。$V^\pi(s)=\mathbb{E}_\pi[G_t\mid S_t=s]$ という「収益の期待」の定義と、ベルマン方程式という「再帰的整合条件」が、同じ価値関数を指していることが数値で確認できました。**(2) 価値の地形に意味がある。** ゴール（右下、価値 0）に近いほど価値が高く（$-5.76$）、遠い左上ほど低い（$-9.36$）。報酬は **どこでも一律 $-1$** なのに、価値は場所でくっきり違う —— これが「報酬 $R$ と価値 $V$ は別物」の数値的な証拠です。$-9.36$ は「左上からランダムに歩くと、平均してこれだけの累積マイナスを払ってゴールに着く」という意味です。

次に、$Q^\pi$ を計算し、$V^\pi(s)=\sum_a \pi(a\mid s)Q^\pi(s,a)$ の関係と「方策改善の芽」を確かめます。

```python title="gridworld_q.py（厳密 V を再利用）"
# V（前スクリプトの V_exact）は計算済みとする。Q^π(s,a)=r+γV(s')（決定論遷移）。
s0 = 0  # 左上スタート
print(f"状態 s0=左上(0) での Q^pi(s0, a):")
for a, name in zip(range(4), ["上", "下", "左", "右"]):
    sn, r = step(s0, a)
    q = r + GAMMA * V_exact[sn]
    print(f"  a={name}: Q = {q:.2f}  (次状態 s'={sn})")
print(f"V^pi(s0) = {V_exact[s0]:.2f}  (= 4 行動の Q の平均)")
```

```text title="出力"
状態 s0=左上(0) での Q^pi(s0, a):
  a=上: Q = -9.42  (次状態 s'=0)
  a=下: Q = -9.30  (次状態 s'=4)
  a=左: Q = -9.42  (次状態 s'=0)
  a=右: Q = -9.30  (次状態 s'=1)
V^pi(s0) = -9.36  (= 4 行動の Q の平均)
```

ここに RL の核心が凝縮しています。**(1) $V$ は $Q$ の方策平均**：$\frac{1}{4}(-9.42-9.30-9.42-9.30)=-9.36=V^\pi(s_0)$。定義式 $V^\pi(s)=\sum_a\pi(a\mid s)Q^\pi(s,a)$ がぴったり成立。**(2) 方策改善の芽**：壁に向かう「上・左」（その場に留まり $-1$ を払うだけ）の $Q$ は $-9.42$、ゴール方向の「下・右」は $-9.30$ で **後者が高い**。もしランダム方策をやめ、各状態で **$Q$ が最大の行動を貪欲に選ぶ** ように変えれば、方策はランダムより必ず良くなる。これが次章 **動的計画法（方策反復・価値反復）** の出発点です。「価値を測る → 価値の高い行動へ方策を寄せる」を繰り返すと、最適方策へ収束していきます。

:::note[学習時 vs 実行（推論）時]
ここで「学習」と「実行」を分けておきます。**学習時**：軌跡を集めて $V^\pi / Q^\pi$ を推定し、それを使って方策を改善する（上のコードの推定パート）。**実行（推論）時**：学習済みの方策 $\pi$ を固定し、状態を見て行動を出すだけ（$Q$ の再推定はしない）。今回の gridworld は環境（$P, R$）が既知なので厳密に解けますが、実際の RL では環境が未知で、軌跡サンプルからしか $V/Q$ を推定できません —— だからモンテカルロや TD 学習（次章以降）が要るのです。
:::

## 演習

::::question[演習 1: 報酬と価値、そして収益の計算]
$\gamma=0.9$ の gridworld で、ある軌跡が「$-1, -1, -1$ の3歩でゴール」だったとします。(a) このときの収益 $G_0$ を計算してください（小数第3位まで）。(b) 「ゴールの1つ手前のマス（状態14）の価値は、毎歩の報酬 $-1$ より高い（0 に近い）」のはなぜですか。報酬と価値の違いに触れて説明してください。(c) 実装出力で左上 $V^\pi=-9.36$、ゴール隣 $V^\pi\approx-5.76$ と大きく違うのに、報酬はどこでも $-1$ でした。これは矛盾しませんか。

:::details[解答]
(a) 収益は $G_0 = R_1 + \gamma R_2 + \gamma^2 R_3 = -1 + 0.9\cdot(-1) + 0.81\cdot(-1) = -1 - 0.9 - 0.81 = \mathbf{-2.710}$。割引のおかげで、3歩ぶんの $-3$ より絶対値が小さくなります（遠い損ほど軽く見る）。

(b) **報酬は即時の1スカラー、価値は将来全体の期待収益** だからです。状態14はゴールに隣接し、ランダム方策でも比較的早く（少ない $-1$ で）ゴールに着けます。価値は「ここから先で払う割引報酬の総和の期待」なので、残り歩数が少ない＝払う $-1$ が少ない状態ほど 0 に近づきます。報酬（その一歩の点数）と価値（残り全部の見込み）を混同しないこと。

(c) 矛盾しません。**報酬が一律でも、価値は「ゴールまでの遠さ」を反映して場所ごとに変わります。** 左上はゴールから遠く、ランダムに歩くと多くの $-1$ を積み上げてからゴールするので価値が低い（$-9.36$）。ゴール隣は少ない $-1$ で済むので価値が高い（$-5.76$）。価値関数は、一律の即時報酬から「状態の有利不利」を浮かび上がらせる装置です。
:::
::::

::::question[演習 2: マルコフ性と状態設計]
ロボットが1次元の直線上を動き、状態を「現在位置 $x$」だけにしたとします。物理的には、次の位置は現在位置と **速度** にも依存します。(a) この「位置だけ」の状態でマルコフ性は満たされますか。(b) 満たされないとしたら、どう状態を設計し直せばよいですか。(c) この問いは「マルコフ性が成り立つかは本質的に何の問題か」という一般的な教訓に繋がります。それは何ですか。

:::details[解答]
(a) **満たされません。** 同じ位置 $x$ でも、速いとき／遅いとき／逆向きのときで次の位置が変わります。次状態の分布が「現在の状態（位置だけ）」では決まらず、隠れた速度に依存してしまうからです。

(b) **状態に速度を含めて $(x, v)$ にする** と回復します。位置と速度の両方が分かれば（物理法則のもとで）次状態の分布は現在状態だけで決まり、マルコフ性が成立します。より一般には「未来予測に必要な情報を状態に全部入れる」。

(c) **マルコフ性が成り立つかどうかは、本質的に「状態をどう設計したか」の問題** です。マルコフ性は環境の固有の性質ではなく、状態に十分な情報を詰め込めるかにかかっています。だから本文でも「過去が無関係なのではなく、必要な過去は状態に畳み込む」と述べました。LLM が全文脈を状態に入れてマルコフ的に振る舞うのと同じ発想です。
:::
::::

## まとめ

:::success[この章の要点]
- 強化学習は **エージェント-環境ループ**（状態 $S$・行動 $A$・報酬 $R$）で、報酬の累積を最大化する **方策 $\pi$** を試行錯誤から学ぶ枠組み。RL は「モダリティ」ではなく **横断する学習パラダイム** で、言語の RLHF・身体性の制御から共通言語として参照される。
- **収益 $G_t = \sum_{k\ge0}\gamma^k R_{t+k+1}$** は将来の割引報酬の総和（確率変数）。**割引率 $\gamma$** は将来をどれだけ重く見るか・無限和を収束させる役。
- **価値関数**は収益の期待値：状態価値 $V^\pi(s)=\mathbb{E}_\pi[G_t\mid S_t=s]$、行動価値 $Q^\pi(s,a)$。両者は $V^\pi(s)=\sum_a\pi(a\mid s)Q^\pi(s,a)$ で結ばれる。
- **報酬 $R$（即時の1スカラー）と価値 $V$（将来全体の期待）は別物。** 一律の報酬でも価値は状態ごとに違う。gridworld で数値的に確認した。
- 問題設定は **MDP $(\mathcal{S},\mathcal{A},P,R,\gamma)$** にまとまる。**マルコフ性**は「現在の状態が未来予測に十分」—— 成り立つかは本質的に **状態設計** の問題。
- 価値は **ベルマン期待方程式**（即時報酬 + 割り引いた次状態の価値）を満たす。これが次章以降のアルゴリズムの土台。
:::

### 次に学ぶこと

この章で **問題（MDP）と評価軸（価値関数・ベルマン方程式）** が手に入りました。実装で見たように、価値を測れば「$Q$ の高い行動へ方策を寄せる」だけで方策が改善できそうだと分かりました。次章では、環境（$P, R$）が既知のときに **ベルマン方程式を反復的に解いて最適方策を求める** 方法 —— **動的計画法（価値反復・方策反復）** に進みます。「価値を測る ↔ 方策を改善する」のループを定式化します。

## 用語ミニ辞典

| 用語 | 一言 |
| --- | --- |
| エージェント / 環境 | 行動を出す側 / 状態と報酬を返す側 |
| 状態 $S_t$ | いま環境がどうなっているか（意思決定に使う情報） |
| 行動 $A_t$ | エージェントが打つ手（離散 or 連続） |
| 報酬 $R_{t+1}$ | 行動の直後にもらう即時のスカラー評価 |
| 方策 $\pi(a\mid s)$ | 状態から行動を選ぶ規則（確率分布） |
| 収益 $G_t$ | 時刻 $t$ 以降の割引報酬の総和（確率変数） |
| 割引率 $\gamma$ | 将来の報酬をどれだけ重く見るか（$0\le\gamma\le1$） |
| 状態価値 $V^\pi(s)$ | $s$ から $\pi$ で得る収益の期待値 |
| 行動価値 $Q^\pi(s,a)$ | $s$ で $a$ を取り以後 $\pi$ に従う収益の期待値 |
| MDP | $(\mathcal{S},\mathcal{A},P,R,\gamma)$。逐次決定問題の標準的な器 |
| 遷移確率 $P(s'\mid s,a)$ | 環境のダイナミクス（次状態の分布） |
| マルコフ性 | 現在の状態が未来予測に十分（過去は状態に畳み込む） |
| ベルマン期待方程式 | 価値 = 即時報酬 + 割引×次状態の価値（自己無撞着） |

## 次のアクション

理論を手で定着させる。**最小の写経 → 動かす → 小実験** を1セットで。

1. 本文の `gridworld_mdp.py` を写経し、`uv run --with numpy python gridworld_mdp.py` で実行する。モンテカルロ推定が厳密値に一致することを自分の目で確認する。
2. **$\gamma$ を変える小実験**：$\gamma$ を $0.5 / 0.9 / 0.99$ に変えて $V^\pi$ がどう変わるか観察する。$\gamma$ が小さいと近視眼的（遠いゴールの価値が効きにくい）になり、価値の差が縮むはずです。
3. **方策改善の芽を確かめる**：各状態で $Q^\pi(s,a)=r+\gamma V(s')$ を全行動について計算し、$Q$ が最大の行動を選ぶ「貪欲方策」を作る。ランダム方策より左上の価値が改善することを確認し、次章の動的計画法へつなぐ。
