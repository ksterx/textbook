# 執筆ガイド（AUTHORING）

このテキストを **分野が変わっても同じ構成・同じ品質** で書くための規約です。
新しい章・分野を書くときは、人間も Claude もこのガイドに従います。
お手本となる完成章: [`docs/audio/01-digital-audio-basics.md`](docs/audio/01-digital-audio-basics.md)。

---

## 1. 全体の方針

- **わかりやすさが最優先。** 迷ったら、正確さを保ったまま、より易しい説明を選ぶ。
- **直感 → 理論 → 数式 → 実装 の順で降りていく。** いきなり数式から始めない。
- **天下りを避ける。** 重要な式は導出する。結論だけを置かない。
- **手を動かせる。** すべての章に、実行可能なコードと演習を入れる。
- **一から積み上げる。** 前提知識を明示し、既習章にリンクする。読者は前提を持たないと仮定する。

## 2. 言語とトーン

- 本文は **日本語**。専門用語は **英語のまま**（例: gradient, attention, sampling）。初出時のみ「日本語訳（English）」または「English（日本語訳）」で併記する。
- 文体は **ですます調**。読者に語りかけるが、馴れ馴れしくしない。
- 1 文は短く。1 段落は 1 つの考え。
- 比喩は直感の節でのみ使い、理論・数式の節では正確な記述を優先する。

## 3. 章の構成（フルセット・順序固定）

すべての章は次の見出しを **この順序** で持つ。テンプレート: [`templates/chapter-template.md`](templates/chapter-template.md)。

| 見出し | 内容 | レベル |
| --- | --- | --- |
| （冒頭）学習目標 | `!!! abstract "学習目標"` の箇条書き。測定可能な動詞で。 | — |
| 前提知識 | 必要な既習事項。既習章にリンク。 | `##` |
| 直感 | なぜ学ぶか・何の問題を解くか。数式を出さない。 | `##` |
| 理論 | 定義・概念・定理を正確に。 | `##` |
| 数式の導出 | ステップを追って導く。結論に $\blacksquare$。 | `##` |
| 実装 | 実行可能なコードで確かめる。 | `##` |
| 演習 | `!!! question` ＋ 折りたたみ解答 `??? success`。 | `##` |
| まとめ | `!!! success "この章の要点"` ＋「次に学ぶこと」。 | `##` |
| 参考文献 | 番号付きリスト。原論文・定番教科書。 | `##` |

- 見出しレベルは `#`（章タイトル）→ `##`（節）→ `###`（小節）。`#` は 1 章に 1 つ。
- セクションは省略せず、内容が薄いときは「削る」のではなく「短くする」。

## 4. 数式の規約

- インライン数式は `$ ... $`、独立行の数式は `$$ ... $$`（前後に空行）。arithmatex(generic) ＋ MathJax で描画される。
- 変数は斜体（`$x$`）、ベクトル・行列は太字（`$\mathbf{x}$`, `$\mathbf{W}$`）。
- 単位やラベルは `\text{}` または `\mathrm{}` で立体に（例: `$f_s\ [\text{Hz}]$`）。
- 導出の各ステップに 1 行の言葉の説明を添える。証明・導出の終わりは `$\blacksquare$`。
- 定義・定理は `!!! note "定理"` などの admonition で囲む。

## 5. コードの規約

- 言語は原則 **Python**。最小の依存（標準ライブラリ＋ NumPy）で動くことを優先する。重い依存（PyTorch 等）は分野の後半でのみ使い、章の前提知識に明記する。
- コードブロックには ` ```python title="<filename>.py" ` のように **title** を付ける。
- **そのまま実行できる** コードにする。擬似コードにしない。
- 期待される出力を別の ` ```text title="出力" ` ブロックで併記する。
- コメントは日本語。何をしているかではなく **なぜそうするか** を書く。

## 6. 図とテーブル

- 図は **Mermaid**（`flowchart`, `sequenceDiagram` 等）。手書きの ASCII 図は使わない。
- 表は Markdown のテーブル。
- 状態を示す絵文字は統一する: ✅ 公開 / 🚧 予定 / 🛠 執筆中。

## 7. 使ってよい Zensical / Markdown 機能（検証済み）

このテキストで使う機能は次に限定する（`zensical.toml` で有効化済み・ビルドで描画確認済み）。

- Admonitions: `!!! note`, `!!! abstract`, `!!! success`, `!!! question`, `!!! warning` など
- 折りたたみ: `??? success "解答"`（演習の解答に使う）
- コードブロック: シンタックスハイライト、`title=`、`hl_lines=`、コピー・行選択ボタン
- 数式: arithmatex(generic) ＋ MathJax
- 図: Mermaid（superfences の custom fence）
- 絵文字: twemoji の **Unicode 絵文字**（🎵 🤖 など）
- グリッドカード: `<div class="grid cards" markdown>`（トップページのみ）

!!! warning "避けるもの"

    - `:material-xxx:` `:octicons-xxx:` などの **アイコン用ショートコード**（描画が不確実）。本文では Unicode 絵文字か、矢印 `→` などで代替する。なお frontmatter の `icon: lucide/...`（ページアイコン）は使ってよい。
    - 新しい依存ライブラリや Markdown 拡張を、`zensical.toml` を更新せずに使うこと。

## 8. ディレクトリと命名

```text
docs/
  index.md                     # トップ（全体マップ）
  <domain>/
    index.md                   # 分野ランディング（ロードマップ）
    NN-<slug>.md               # 章。NN は 2 桁ゼロ詰め（01, 02, ...）、slug は英小文字ハイフン区切り
```

- 分野ディレクトリ名は **英小文字ハイフン区切り**（例: `reinforcement-learning`）。
- 章ファイルは `NN-<slug>.md`。番号は学ぶ順序を表す。

## 9. 新しい章を追加する手順

1. [`templates/chapter-template.md`](templates/chapter-template.md) を `docs/<domain>/NN-<slug>.md` にコピーする。
2. プレースホルダをすべて埋める（このガイドの規約に従う）。
3. `zensical.toml` の `nav` の、その分野のセクション配列に 1 行追加する:
   `{ "N. <章タイトル>" = "<domain>/NN-<slug>.md" },`
4. `docs/<domain>/index.md` の「章一覧」テーブルの状態を ✅ に更新し、リンクを張る。
5. `uv run zensical build` でビルドが通ることを確認する。

## 10. 新しい分野を追加する手順

1. `docs/<domain>/` を作り、[`templates/domain-index-template.md`](templates/domain-index-template.md) を `docs/<domain>/index.md` にコピーして埋める。
2. `zensical.toml` の `nav` にトップレベルのセクションを 1 つ追加する:
   ```toml
   { "<分野名（日本語）>" = [
     "<domain>/index.md",
   ] },
   ```
3. トップページ [`docs/index.md`](docs/index.md) の「分野一覧」カードに 1 枚追加する。
4. 最初の章を「9. 新しい章を追加する手順」に従って書く。
